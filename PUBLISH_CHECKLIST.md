# Checklist de publicación (GitHub + LinkedIn)

## 1) GitHub
- [ ] `README.md` completo con objetivo, stack, instalación, uso y resultados
- [ ] `.gitignore` validado (sin `env/`, logs pesados, artefactos temporales)
- [ ] Repositorio limpio (`git status` sin cambios inesperados)
- [ ] Licencia agregada (MIT recomendada si quieres código abierto)
- [ ] 1 release inicial creada (`v1.0.0`)
- [ ] Evidencia visual en repo (1 GIF/video + 1 gráfica)
- [ ] Sección de limitaciones y trabajo futuro documentada

## 2) Calidad técnica mínima
- [ ] Comandos de entrenamiento y evaluación probados
- [ ] Dependencias instalables desde instrucciones del README
- [ ] Rutas de archivos coherentes (`model/ppo_mario_final.zip`, `model/reward_log.csv`)
- [ ] Scripts principales ejecutan sin errores en entorno limpio

## 3) LinkedIn
- [ ] Captura principal del resultado (gameplay)
- [ ] Gráfica de entrenamiento clara
- [ ] Post con estructura: problema → solución → resultados → aprendizajes
- [ ] Link al repositorio agregado
- [ ] Hashtags técnicos + CTA para networking

## 4) Antes de publicar
- [ ] Revisar ortografía y formato del post
- [ ] Confirmar que no haya secretos/credenciales en el repo
- [ ] Verificar que no haya archivos grandes bloqueados por GitHub
- [ ] Probar el repositorio como si fueras un reclutador (en 60-90 segundos)

## 5) Después de publicar
- [ ] Responder comentarios con contexto técnico
- [ ] Fijar el repositorio en tu perfil de GitHub
- [ ] Añadir el proyecto en la sección de proyectos del CV
- [ ] Guardar métricas de interacción para mejorar el siguiente post
